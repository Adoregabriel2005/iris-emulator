# Atari 2600 (Stella) Technical Instructions

## 1. Hardware Architecture & Constraints
- **CPU:** MOS 6507 (a 6502 subset with 13 address lines). Total addressable space is 8KB, but only 4KB is mapped for cartridges without bankswitching.
- **TIA (Television Interface Adaptor):** The core of the video/audio system. It operates on a "line-by-line" basis.
- **RIOT (6532):** Provides 128 bytes of RAM, I/O for controllers, and a programmable timer.

## 2. Memory Map (Crucial for Emulation)
- **$0000 - $003F:** TIA Registers (Write-only for most, some Read-only for collisions).
- **$0080 - $00FF:** RIOT RAM (128 bytes).
- **$0280 - $029F:** RIOT Registers (Timers and I/O).
- **$F000 - $FFFF:** ROM Space (Standard 4KB).

## 3. TIA Video Timing & Protocol
- **Scanline Cycles:** One horizontal scanline consists of 228 TIA color clocks.
    - 68 clocks: Horizontal Blank (HBLANK).
    - 160 clocks: Active Video.
- **Frame Structure:**
    - 3 lines: VSYNC (Microprocessor controlled).
    - 37 lines: VBLANK.
    - 192 lines: Active Picture.
    - 30 lines: OVERSCAN.
- **Synchronization:** The `WSYNC` (Write $02) register must halt the CPU until the start of the next HBLANK to ensure perfect alignment.

## 4. Graphics Objects & Priority
- **Objects:** Two Players (P0, P1), two Missiles (M0, M1), one Ball (BL), and a Playfield (PF).
- **Playfield:** 20 bits wide, representing half the screen. Can be duplicated or reflected via `CTRLPF`.
- **Priority Encoder:** Normal priority is P0/M0 > P1/M1 > PF/BL > BK (Background). This can be modified by bit 2 of `CTRLPF`.
- **Collisions:** 15 possible two-object collisions detected in hardware. Read from registers $00 to $0F.

## 5. Movement and Positioning
- **Horizontal:** Use `RESPx` to reset position and `HMOVE` to apply fine motion offsets stored in `HMxx` registers.
- **Vertical:** There is NO hardware vertical positioning. The CPU must enable/disable graphics registers (`GRP0`, `GRP1`, `ENAM0`, etc.) at the exact scanline during the 192-line active window.

## 6. Implementation Rules for Gemini
- **Strict Timing:** All emulated TIA operations must be synchronized with the 6507 clock (3 TIA clocks = 1 CPU clock).
- **Illegal Opcodes:** Support illegal 6502 opcodes (e.g., LAX, SAX) as they are used in high-performance 2600 games.
- **Bankswitching:** When implementing mappers (F8, F6, etc.), intercept reads/writes to specific hotspots (e.g., $1FF8, $1FF9).
